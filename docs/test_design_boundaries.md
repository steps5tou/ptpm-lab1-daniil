# Лабораторная работа 5. Тест-дизайн: границы и эквивалентность

## Цель работы

Выделить классы эквивалентности и граничные значения для полей ввода модуля «Сервис задач», обосновать выбор тестовых значений.

## Связь с предыдущими лабораторными

Поля выбраны на основе docs/requirements.md (R-01, R-03, R-07).
Сценарии использования полей описаны в docs/scenarios.md (S-01, S-03, S-04).
Функциональный чек-лист — docs/functional_checklist.md.
Тест-кейсы — docs/test_cases.md.

## Выбранные поля ввода

1. Название задачи — строковое поле, обязательное.
2. Статус задачи — строковое поле с фиксированным списком значений.
3. Ключевое слово поиска — строковое поле, необязательное.

---

## Поле 1. Название задачи

### Правило ввода

Название задачи должно быть непустой строкой длиной от 1 до 100 символов включительно.

### Классы эквивалентности

| Класс | Описание | Тип |
|-------|----------|-----|
| КЭ-1 | Строка длиной от 1 до 100 символов | Корректный |
| КЭ-2 | Пустая строка | Некорректный |
| КЭ-3 | Строка длиннее 100 символов | Некорректный |
| КЭ-4 | Строка только из пробелов | Некорректный |

### Граничные значения

| Значение | Описание |
|----------|----------|
| 0 символов (пустая строка) | Ниже нижней границы |
| 1 символ | Нижняя граница |
| 2 символа | Сразу после нижней границы |
| 99 символов | Перед верхней границей |
| 100 символов | Верхняя граница |
| 101 символ | Выше верхней границы |

### Тесты

TD-01. Поле: Название задачи. Значение: «Smoke». Ожидаемый результат: задача создана. Обоснование: проверка корректного значения из допустимого класса.

TD-02. Поле: Название задачи. Значение: «A» (1 символ). Ожидаемый результат: задача создана. Обоснование: проверка нижней границы допустимого диапазона.

TD-03. Поле: Название задачи. Значение: пустая строка. Ожидаемый результат: система отклоняет ввод и возвращает ошибку. Обоснование: проверка значения ниже нижней границы.

TD-04. Поле: Название задачи. Значение: строка из 100 символов (100 букв «a»). Ожидаемый результат: задача создана. Обоснование: проверка верхней границы допустимого диапазона.

TD-05. Поле: Название задачи. Значение: строка из 101 символа. Ожидаемый результат: система отклоняет ввод и возвращает ошибку. Обоснование: проверка значения выше верхней границы.

TD-06. Поле: Название задачи. Значение: «   » (только пробелы). Ожидаемый результат: система отклоняет ввод и возвращает ошибку. Обоснование: проверка некорректного класса — строка из пробелов не является допустимым названием.

## Поле 2. Статус задачи

### Правило ввода

Статус задачи должен принимать одно из трёх допустимых значений: «новая», «в работе», «выполнена».

### Классы эквивалентности

| Класс | Описание | Тип |
|-------|----------|-----|
| КЭ-1 | Одно из значений: «новая», «в работе», «выполнена» | Корректный |
| КЭ-2 | Любая другая строка | Некорректный |
| КЭ-3 | Пустая строка | Некорректный |
| КЭ-4 | Число или другой тип данных | Некорректный |

### Граничные значения

Поле относится к списку допустимых значений, а не к числовому диапазону. Граничные значения определяются как крайние элементы списка и значения за его пределами.

| Значение | Описание |
|----------|----------|
| «новая» | Первый допустимый элемент списка |
| «выполнена» | Последний допустимый элемент списка |
| «в работе» | Средний допустимый элемент списка |
| «сделано» | Значение за пределами списка |
| пустая строка | Значение за пределами списка |

### Тесты

TD-07. Поле: Статус задачи. Значение: «новая». Ожидаемый результат: статус принят. Обоснование: проверка первого допустимого значения из списка.

TD-08. Поле: Статус задачи. Значение: «в работе». Ожидаемый результат: статус принят. Обоснование: проверка среднего допустимого значения из списка.

TD-09. Поле: Статус задачи. Значение: «выполнена». Ожидаемый результат: статус принят. Обоснование: проверка последнего допустимого значения из списка.

TD-10. Поле: Статус задачи. Значение: «сделано». Ожидаемый результат: система отклоняет ввод, возвращает ошибку с указанием допустимых значений. Обоснование: проверка значения из некорректного класса — строка не входит в список допустимых.

TD-11. Поле: Статус задачи. Значение: пустая строка. Ожидаемый результат: система отклоняет ввод и возвращает ошибку. Обоснование: проверка пустого значения — недопустимый класс.

TD-12. Поле: Статус задачи. Значение: «НОВАЯ» (верхний регистр). Ожидаемый результат: система отклоняет ввод и возвращает ошибку. Обоснование: проверка чувствительности к регистру — значение не совпадает ни с одним допустимым.

## Поле 3. Ключевое слово поиска

### Правило ввода

Ключевое слово для поиска должно быть непустой строкой. Поиск выполняется без учёта регистра. Если строка пустая, поиск не выполняется.

### Классы эквивалентности

| Класс | Описание | Тип |
|-------|----------|-----|
| КЭ-1 | Непустая строка, присутствующая в данных | Корректный, совпадение найдено |
| КЭ-2 | Непустая строка, отсутствующая в данных | Корректный, совпадений нет |
| КЭ-3 | Пустая строка | Некорректный |
| КЭ-4 | Строка в другом регистре | Корректный, должно найти |

### Граничные значения

| Значение | Описание |
|----------|----------|
| 1 символ | Минимально возможный запрос |
| пустая строка | Ниже минимума |
| слово целиком совпадает с названием | Точное совпадение |
| слово в другом регистре | Проверка регистронезависимости |

### Тесты

TD-13. Поле: Ключевое слово поиска. Значение: «Smoke». Ожидаемый результат: возвращается задача с названием «Smoke». Обоснование: проверка корректного значения из допустимого класса — точное совпадение.

TD-14. Поле: Ключевое слово поиска. Значение: «smoke» (нижний регистр). Ожидаемый результат: возвращается задача с названием «Smoke». Обоснование: проверка регистронезависимого поиска — класс корректных значений с другим регистром.

TD-15. Поле: Ключевое слово поиска. Значение: «zzz». Ожидаемый результат: возвращается пустой список. Обоснование: проверка корректного класса — слово отсутствует в данных.

TD-16. Поле: Ключевое слово поиска. Значение: пустая строка. Ожидаемый результат: система возвращает ошибку или не выполняет поиск. Обоснование: проверка некорректного класса — пустой запрос.

TD-17. Поле: Ключевое слово поиска. Значение: «S» (1 символ). Ожидаемый результат: возвращаются все задачи, содержащие букву «S». Обоснование: проверка минимальной длины запроса — нижняя граница.

---

## Общий вывод

Наибольший риск ошибок — в поле «Статус задачи», потому что разработчик может допустить ошибку в проверке регистра или не включить все допустимые значения в список. Ошибка здесь приводит к тому, что задача получает некорректный статус или корректный статус отклоняется.

Самые важные тесты — TD-03 (пустое название) и TD-10 (недопустимый статус), потому что именно эти случаи чаще всего встречаются при реальном использовании и именно там разработчики забывают добавить проверку входных данных.
