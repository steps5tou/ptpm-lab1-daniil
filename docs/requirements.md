# Требования к модулю «Сервис задач»

## Описание модуля

Модуль предназначен для управления задачами студента. Модуль обеспечивает создание задач, изменение статуса, просмотр списка, поиск по ключевому слову и хранение данных в файле JSON.

Модуль не отвечает за пользовательский интерфейс, сетевое взаимодействие, регистрацию пользователей и разграничение прав доступа.

## Функциональные требования

R-01: модуль должен позволять создать задачу с названием и описанием. Проверка: после создания задача появляется в списке и имеет идентификатор.

R-02: модуль должен назначать каждой задаче уникальный идентификатор. Проверка: идентификатор существует, не пустой, не повторяется среди других задач.

R-03: модуль должен поддерживать следующие статусы задачи: новая, в работе, выполнена. Проверка: задача принимает только эти три значения статуса.

R-04: модуль должен поддерживать изменение статуса задачи только на допустимые значения. Проверка: при передаче недопустимого статуса статус задачи не меняется и возвращается ошибка.

R-05: модуль должен возвращать полный список задач. Проверка: список содержит все созданные задачи с полями id, title, description, status.

R-06: модуль должен поддерживать фильтрацию списка задач по статусу. Проверка: фильтр возвращает только задачи с указанным статусом.

R-07: модуль должен поддерживать поиск задач по ключевому слову в названии и описании. Проверка: поиск возвращает задачи, содержащие ключевое слово в поле title или description.

R-08: поиск по ключевому слову должен работать независимо от регистра символов. Проверка: поиск по слову «задача» находит задачу с названием «ЗАДАЧА».

R-09: модуль должен сохранять все задачи в файл JSON после каждого изменения. Проверка: файл tasks.json обновляется сразу после создания или изменения задачи.

R-10: модуль должен загружать задачи из файла JSON при инициализации. Проверка: после перезапуска задачи восстанавливаются без потери полей.

R-11: модуль должен поддерживать редактирование названия и описания существующей задачи. Проверка: после редактирования задача возвращает обновлённые поля.

R-12: количество задач не ограничено. Проверка: можно создать более одной задачи без ошибок.

## Требования к данным и формату хранения

R-13: файл данных хранится в корневой папке проекта под именем tasks.json. Проверка: файл создаётся по указанному пути.

R-14: каждая задача содержит обязательные поля id, title, description, status. Проверка: созданная задача имеет все четыре поля.

Пример минимальной записи:

```json
{
  "tasks": [
    {
      "id": "1",
      "title": "Сдать лабораторную",
      "description": "",
      "status": "новая"
    }
  ]
}
```

## Требования к ошибкам и исключениям

R-15: при попытке создать задачу с пустым названием модуль должен вернуть ошибку. Проверка: задача не создаётся, возвращается сообщение об ошибке.

R-16: при передаче недопустимого статуса модуль должен вернуть ошибку с указанием допустимых значений. Проверка: статус не изменяется, возвращается список допустимых значений.

R-17: при обращении к несуществующему идентификатору модуль должен вернуть ошибку. Проверка: возвращается сообщение об отсутствии задачи.

R-18: если файл JSON отсутствует при запуске, модуль начинает работу с пустым списком. Проверка: модуль запускается без ошибок, список задач пуст.

R-19: если файл JSON повреждён, модуль возвращает ошибку и не перезаписывает файл. Проверка: файл остаётся нетронутым, возвращается сообщение о повреждении.

## Ограничения и допущения

Модуль рассчитан на одного пользователя без одновременного доступа. Уникальность задачи определяется идентификатором, а не названием. Данные хранятся только локально в файле JSON.

## Критерии приёмки

1. После создания задача появляется в списке и сохраняется в файле JSON.
2. После перезапуска и загрузки из файла задача сохраняет все поля и статус.
3. Поиск по ключевому слову находит задачу независимо от регистра.
4. Попытка установить недопустимый статус не изменяет текущий статус задачи.
5. При отсутствии файла JSON модуль запускается без ошибок с пустым списком.
6. Фильтрация по статусу возвращает только задачи с указанным статусом.
