# Требования к модулю «Сервис задач»

## Описание модуля

Модуль предназначен для управления задачами студента. Модуль обеспечивает создание задач, изменение статуса, просмотр списка, поиск по ключевому слову и хранение данных в файле JSON.

Модуль не отвечает за пользовательский интерфейс, сетевое взаимодействие, регистрацию пользователей и разграничение прав доступа.

## Функциональные требования

1. Модуль должен позволять создать задачу с названием и описанием.
2. Модуль должен назначать каждой задаче уникальный идентификатор.
3. Модуль должен поддерживать следующие статусы задачи: новая, в работе, выполнена.
4. Модуль должен поддерживать изменение статуса задачи только на допустимые значения.
5. Модуль должен возвращать полный список задач.
6. Модуль должен поддерживать фильтрацию списка задач по статусу.
7. Модуль должен поддерживать поиск задач по ключевому слову в названии и описании.
8. Поиск по ключевому слову должен работать независимо от регистра символов.
9. Модуль должен сохранять все задачи в файл JSON после каждого изменения.
10. Модуль должен загружать задачи из файла JSON при инициализации.
11. Модуль должен поддерживать редактирование названия и описания существующей задачи.
12. Количество задач не ограничено.

## Требования к данным и формату хранения

Файл данных хранится в корневой папке проекта под именем tasks.json.

Каждая задача содержит следующие обязательные поля:

- id — строка, уникальный идентификатор задачи
- title — строка, название задачи, не может быть пустым
- description — строка, описание задачи, может быть пустым
- status — строка, одно из значений: новая, в работе, выполнена
