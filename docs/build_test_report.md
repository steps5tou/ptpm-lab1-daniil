## Блокирующие дефекты

Выпуск сборки блокируется следующими дефектами.

**Issue #15 — При первом запуске без файла данных программа падает (Critical)**
Делает модуль полностью неработоспособным при первом развёртывании или переносе на новое устройство. Аварийное завершение происходит до начала основной работы.

**Issue #13 — Повреждённый JSON завершает программу без сообщения (High)**
При наличии повреждённого файла данных модуль аварийно завершается, что делает восстановление после ошибки невозможным без ручного вмешательства.

**Issue #11 — Падение программы при пустом названии задачи (High)**
Основной сценарий создания задачи завершается аварийно при некорректном вводе вместо вывода понятного сообщения об ошибке.
