# Тест-кейсы модуля «Сервис задач»

## Шаблон тест-кейса

- Идентификатор: TC-XX
- Название: краткое описание проверки
- Требование: R-XX
- Сценарий: S-XX
- Предусловия: состояние системы до начала теста
- Входные данные: конкретные значения
- Шаги: пронумерованные действия
- Ожидаемый результат: наблюдаемый итог
- Постусловия: состояние системы после теста
- Важность: BLOCK или WARN

## Набор данных

Задача A: название «Smoke», описание «Release» — короткая задача.
Задача B: название «Отчёт по практике», описание «Включить все разделы: введение, основная часть, заключение, список литературы, приложения» — задача с длинным описанием.
Задача C: название «PyTest Case», описание «проверка поиска без учёта регистра» — для проверки регистронезависимого поиска.
Ошибочный ввод D: пустая строка в качестве названия задачи.
Ошибочный ввод E: статус «сделано» — недопустимое значение.

## Тест-кейсы

### TC-01: создание задачи с валидными полями
- Требование: R-01, R-02
- Сценарий: S-01
- Предусловия: модуль инициализирован, файл tasks.json отсутствует или пуст.
- Входные данные: задача A — название «Smoke», описание «Release».
- Шаги:
  1. Вызвать функцию создания задачи с названием «Smoke» и описанием «Release».
  2. Вызвать функцию получения списка задач.
- Ожидаемый результат: задача присутствует в списке, имеет уникальный идентификатор и статус «новая».
- Постусловия: файл tasks.json содержит одну запись.
- Важность: BLOCK
- Защита от регрессии: если при рефакторинге изменится логика назначения идентификатора или начального статуса, этот тест сразу покажет отклонение.
