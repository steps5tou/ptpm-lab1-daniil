## Задание для закрепления

### Выбранный дефект: Дефект 5 — падение при первом запуске без файла данных

### Почему нельзя считать мелочью
Этот дефект блокирует работу модуля при первом запуске или переносе проекта на новое устройство. Пользователь не может начать работу вообще, что делает продукт неработоспособным в базовом сценарии.

### Что может пойти не так без ретеста
Разработчик может исправить только один конкретный путь к файлу, но не учесть другие случаи отсутствия файла. Без ретеста дефект может остаться в смежных сценариях.

### Тест для предотвращения повторения
Добавить тест-кейс TC-10 в регрессионный набор: при каждом запуске проверять, что модуль инициализируется без ошибок при отсутствии файла tasks.json.
