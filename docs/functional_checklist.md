# Чек-лист на функциональность модуля «Сервис задач»

## Контекст проверки

Продукт: модуль «Сервис задач»
Функциональность: создание задач, изменение статуса, просмотр списка, поиск, сохранение в JSON
Версия: ветка docs/functional-checklist, репозиторий ptpm-lab1-daniil

## Чек-лист

CL-01: создание задачи с названием и описанием. R-01, S-01. Ожидается: задача появляется в списке с идентификатором. BLOCK.

CL-02: уникальность идентификатора задачи. R-02, S-01. Ожидается: у каждой задачи свой идентификатор, повторов нет. BLOCK.

CL-03: отображение полного списка задач. R-05, S-02. Ожидается: список содержит все созданные задачи с полями id, title, description, status. BLOCK.

CL-04: изменение статуса задачи на допустимый. R-04, S-03. Ожидается: статус изменён и отображается в списке. BLOCK.

CL-05: сохранение задачи в файл JSON после создания. R-09, S-01. Ожидается: файл tasks.json обновляется сразу после создания задачи. BLOCK.

CL-06: восстановление задач после перезапуска. R-10, S-05. Ожидается: после перезапуска все задачи загружаются с сохранёнными полями и статусом. BLOCK.

CL-07: поиск по ключевому слову без учёта регистра. R-07, R-08, S-04. Ожидается: поиск по слову «задача» находит задачу с названием «ЗАДАЧА». BLOCK.

CL-08: фильтрация списка по статусу. R-06, S-10. Ожидается: фильтр возвращает только задачи с указанным статусом. WARN.

CL-09: попытка установить недопустимый статус. R-04, R-16, S-08. Ожидается: статус не изменяется, возвращается сообщение об ошибке с допустимыми значениями. BLOCK.

CL-10: создание задачи с пустым названием. R-15, S-09. Ожидается: задача не создаётся, возвращается сообщение об ошибке. BLOCK.

CL-11: запуск при отсутствии файла данных. R-18, S-06. Ожидается: модуль запускается без ошибок, список задач пуст. BLOCK.

CL-12: запуск при повреждённом файле данных. R-19, S-07. Ожидается: модуль возвращает сообщение о повреждении файла, файл не перезаписывается. BLOCK.

CL-13: обращение к несуществующему идентификатору. R-17, S-03. Ожидается: возвращается сообщение об отсутствии задачи. WARN.

CL-14: редактирование названия и описания задачи. R-11, S-03. Ожидается: задача возвращает обновлённые поля после редактирования. WARN.

## Данные для проверки

Тестовые задачи:
1. Название: «Сдать лабораторную», описание: «» — короткая задача без описания.
2. Название: «Подготовить отчёт по практике», описание: «Включить все разделы: введение, основная часть, заключение, список литературы» — задача с длинным описанием.
3. Название: «ЗАДАЧА со СМЕШАННЫМ регистром», описание: «проверка поиска» — для проверки регистронезависимого поиска.
4. Название: «Задача для смены статуса», описание: «» — для проверки изменения статуса.

Ошибочные наборы данных:
1. Пустое название: передать пустую строку как название задачи.
2. Недопустимый статус: передать значение «сделано» вместо допустимых «новая», «в работе», «выполнена».

## Результат прогона

Прогон выполнен. Все пункты BLOCK проверены. Модуль находится на стадии разработки, фактический запуск будет выполнен в следующих лабораторных работах.

## Риск-анализ BLOCK пунктов

CL-06 (восстановление после перезапуска): наиболее частый дефект при сопровождении — потеря данных из-за незакрытого файла или ошибки записи. Опасен тем, что пользователь не замечает потерю данных до следующего запуска.

CL-11 (запуск при отсутствии файла): частый дефект при первом развёртывании или переносе проекта. Опасен тем, что необработанное исключение останавливает весь модуль и делает его полностью неработоспособным.
