# Лабораторная работа 7. Воспроизведение и локализация дефекта

## Общие сведения

**Дисциплина:** Поддержка и тестирование программных модулей
**Студент:** Даниил
**Проект:** Сервис задач

---

## Выбранный дефект

**Название:** При первом запуске без файла данных программа падает
**Ссылка на Issue:** https://github.com/steps5tou/ptpm-lab1-daniil/issues/15
**Краткое описание:** При запуске модуля в отсутствие файла tasks.json программа аварийно завершается с исключением FileNotFoundError вместо того чтобы начать работу с пустым списком задач.

---

## Нарушенное требование и сценарий

По требованию R-18 модуль должен запускаться без ошибок при отсутствии файла tasks.json и начинать работу с пустым списком задач. По сценарию S-06 система должна корректно обработать отсутствие файла и не завершать работу аварийно. Фактическое поведение нарушает оба артефакта.

---

## Окружение

Python 3.14, Windows 10, запуск из терминала PyCharm, ветка main, проект ptpm-lab1-daniil.

---

## Предусловия

Файл tasks.json отсутствует в каталоге проекта. Виртуальное окружение .venv активировано. Программа ранее не запускалась в данном каталоге.

---

## Шаги воспроизведения

1. Убедиться, что файл tasks.json отсутствует в каталоге проекта.
2. Открыть терминал в корневой папке проекта.
3. Активировать виртуальное окружение командой `.venv\Scripts\activate`.
4. Запустить модуль командой `python src/app.py`.
5. Наблюдать результат запуска.

## Фактический результат

Программа аварийно завершается с исключением FileNotFoundError. Сообщение об ошибке указывает на попытку открыть файл tasks.json, который не существует. Работа модуля не начинается.

## Ожидаемый результат

Согласно требованию R-18 модуль должен запуститься без ошибок, вывести пустой список задач и создать файл tasks.json при первом сохранении.

---

## Вывод о воспроизводимости

Дефект воспроизводится стабильно при каждом повторе в одинаковом окружении. Достаточно удалить файл tasks.json и запустить программу — результат одинаков во всех случаях. Дефект не является случайным или нестабильным.

---

## Локализация дефекта

Проблема не связана с логикой меню, созданием задач или поиском — эти части программы не успевают выполниться. Аварийное завершение происходит в момент инициализации модуля, до начала основной работы.

Локализация указывает на модуль загрузки данных, вероятнее всего на функцию, которая открывает файл tasks.json при старте. Проблема находится в участке кода, где выполняется чтение файла — отсутствует проверка существования файла перед его открытием и отсутствует обработка исключения FileNotFoundError.

Уровень проблемы: обработка исключений при инициализации, чтение файла данных.
Предполагаемый модуль: функция загрузки задач из файла JSON.
Предполагаемый участок: вызов open() или json.load() без проверки наличия файла.

---

## Предполагаемая причина

Симптом: программа аварийно завершается при запуске.
Локализация: функция загрузки данных из файла tasks.json.
Предполагаемая причина: отсутствует проверка существования файла перед его открытием. Вместо условия «если файл существует — загрузить, иначе — начать с пустым списком» код пытается открыть файл напрямую и не обрабатывает исключение FileNotFoundError.

---

## Итоговый вывод

Проанализирован дефект падения программы при первом запуске без файла данных. Дефект воспроизводится стабильно при каждом повторе в одинаковом окружении. Локализация указывает на модуль загрузки задач и участок открытия файла tasks.json. В первую очередь необходимо проверить функцию инициализации данных и добавить обработку случая отсутствия файла с возвратом пустого списка задач.

---

## Почему нельзя исправлять дефект без локализации

Если разработчик начнёт вносить изменения, не понимая где именно находится проблема, он рискует исправить случайный симптом, а не причину. Например, он может добавить проверку в одном месте, тогда как та же ошибка возникает в другом участке кода при схожих условиях. Без локализации невозможно написать точный ретест, потому что неясно, что именно проверять после исправления. В итоге дефект может вернуться в следующей версии, и цикл повторится заново.
